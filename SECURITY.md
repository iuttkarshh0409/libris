# Security Policy

We take security issues seriously. If you discover a vulnerability or security issue in Libris, please follow the guidelines below to report it.

---

## 1. Supported Versions

Only the current main release version of the platform is actively supported with security updates:

| Version | Supported |
| --- | --- |
| v1.0.x | Yes |
| < v1.0.0 | No |

---

## 2. Reporting a Vulnerability

**Please do not report security vulnerabilities via public GitHub issues.** 

Instead, send a detailed report to the project maintainers.

Please include the following details in your report:
*   A description of the vulnerability and its potential impact.
*   Step-by-step instructions (or a proof-of-concept script) to reproduce the issue.
*   Details of the environment in which you reproduced the bug.

We will acknowledge receipt of your vulnerability report within 48 hours and work with you to coordinate a security fix and release.
