# Future Roadmap

> "The Future Roadmap outlines the long-term evolution of Libris. It organizes planned capabilities into strategic horizons, ensuring that future development remains purposeful, incremental, and aligned with the platform's vision."

---

# 1. Roadmap Philosophy

The roadmap is divided into three horizons.

Each horizon represents a distinct stage in the platform's maturity.

Features should only be considered for implementation after the objectives of the current horizon have been successfully achieved.

The roadmap is intended to guide strategic direction rather than define strict release schedules.

---

# 2. Horizon 1 — Complete the Platform

## Objective

Deliver a stable, trustworthy, and fully functional academic Knowledge Retrieval Platform.

This horizon focuses on completing the core capabilities required for everyday study and reference.

---

### Knowledge Management

- Multi-book library
- Book metadata management
- Knowledge Index management
- Index rebuilding
- Configuration management

---

### Retrieval

- Semantic retrieval
- Context assembly
- Citation generation
- Supporting excerpts
- Retrieval transparency

---

### User Experience

- Dashboard
- Query Workspace
- Evidence Explorer
- Progress indicators
- Error reporting

---

### Reliability

- Comprehensive logging
- Robust error handling
- Configuration validation
- Health monitoring

---

### Documentation

- Complete architecture documentation
- API documentation
- ADR history
- Evaluation framework

---

## Success Criteria

The platform should become a reliable daily companion for textbook-based study.

---

# 3. Horizon 2 — Intelligent Retrieval

## Objective

Improve retrieval quality without expanding the platform's knowledge boundary.

This horizon focuses on retrieval engineering rather than additional AI capabilities.

---

### Retrieval Improvements

- Metadata-aware ranking
- Cross-encoder reranking
- Adaptive retrieval depth
- Query expansion
- Multi-query retrieval

---

### Knowledge Processing

- OCR support
- Table extraction
- Figure extraction
- Diagram indexing
- Formula preservation

---

### Context Optimization

- Dynamic context assembly
- Duplicate elimination
- Context compression
- Evidence prioritization

---

### Explainability

- Retrieval visualization
- Similarity score inspection
- Chunk relationship graph
- Evidence confidence reporting

---

### Performance

- Faster indexing
- Incremental indexing
- Parallel processing
- Index optimization

---

## Success Criteria

The platform should retrieve higher-quality evidence while preserving transparency and determinism.

# 5. Ideas Under Evaluation

The following ideas require further research before adoption.

- Graph-based retrieval
- Agent-assisted retrieval workflows
- Multi-modal retrieval
- Voice interaction
- Handwritten note indexing
- Collaborative study spaces
- Federated knowledge indexes

These ideas should not be implemented without supporting architectural decisions and evaluation.

---

# 6. Explicitly Out of Scope

The platform intentionally avoids capabilities that conflict with its educational philosophy.

Examples include:

- General-purpose internet search
- Ungrounded AI conversations
- Automatic assignment completion
- Examination answer generation
- Fabricated citations
- Knowledge without evidence

These exclusions preserve the platform's commitment to trustworthy, evidence-based learning.

---

# 7. Roadmap Review

The roadmap is a living document.

New ideas should be evaluated against:

- Project Vision
- Product Principles
- Guardrails
- System Architecture

Accepted changes should be accompanied by an Architecture Decision Record (ADR) where appropriate.

---

# Roadmap Summary

The long-term vision is not to build a more intelligent chatbot.

It is to build a more trustworthy academic Knowledge Retrieval Platform.

Each horizon strengthens the platform by improving retrieval quality, transparency, and the overall learning experience while remaining grounded in verifiable academic sources.
